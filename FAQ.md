A: The app is free if you use open-source LLMs (like Llama or Mixtral). Using Azure LLMs requires payment.

A: Redis caches answers to speed up repeated questions.

**Q: What is a dense vector?**  
A: A dense vector is a numerical representation of text (like a sentence or document) generated by a neural network (embedding model). It captures the semantic meaning of the text, allowing for similarity search based on meaning rather than exact words.

---

**Q: What is a sparse vector?**  
A: A sparse vector is a representation of text where most values are zero, and only a few positions (corresponding to specific words or terms) have non-zero values. In search, sparse vectors are often used for keyword-based methods like BM25, focusing on exact word matches.

---

**Q: What is semantic search?**  
A: Semantic search uses dense vectors (embeddings) to find documents that are similar in meaning to a query, even if they don’t share the same exact words.

---

**Q: What is hybrid search?**  
A: Hybrid search combines both dense (semantic) and sparse (keyword/BM25) search methods. It retrieves results from both approaches and fuses them (e.g., using Reciprocal Rank Fusion) to leverage the strengths of each.

---

**Q: What is BM25?**  
A: BM25 is a popular keyword-based ranking function used in information retrieval. It scores documents based on the presence and frequency of query terms, favoring exact matches.


---

**Q: What is RRF Fusion?**  
A: RRF (Reciprocal Rank Fusion) is a method to combine ranked lists from multiple search strategies (e.g., dense and sparse). For each result, you assign a score: 1 / (rrf_k + rank), where rank is its position in the list and rrf_k is a constant (usually a large number to smooth scores). You sum these scores across all lists for each document. The final ranking is based on the total fused score. RRF is simple, robust, and doesn’t require tuning. It helps merge results from different retrieval modes (semantic and keyword) so you get the best of both worlds.

---

**Q: What is Qdrant DB?**  
A: Qdrant is an open-source vector database designed for storing and searching embeddings (vectors). It supports both dense (vector/embedding) and sparse (keyword/BM25) search. It’s fast, scalable, and supports hybrid search natively. You can store documents with both embeddings and sparse vectors, and query them using either or both.

---

**Q: Can we use FAISS instead of Qdrant?**  
A: FAISS is another popular vector search library (by Facebook/Meta). It’s optimized for dense vector search (embeddings). FAISS does not natively support sparse search (BM25/keyword) or hybrid search. If you only need semantic search, FAISS is a good choice. For hybrid (dense + sparse), Qdrant is better.

---

**Q: Is sparse and dense search an option with Qdrant?**  
A: Yes! Qdrant supports both dense (vector) and sparse (BM25/keyword) search. You can store both types of representations for each document. You can query using either or both, and Qdrant will return results accordingly.

---

**Q: Can we pass BM25 and embedding to Qdrant? Does Qdrant have such compatibility?**  
A: Yes, Qdrant is designed for this. You can store both embeddings (for dense search) and BM25 vectors (for sparse search) in Qdrant. When querying, you can specify which mode to use (using="dense", using="sparse", or both). Qdrant can perform hybrid search and fuse results (like with RRF).
